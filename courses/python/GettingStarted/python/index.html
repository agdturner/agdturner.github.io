Python

Introduction

Python is a popular and powerful interpretted programming language. The Python interpreter and the extensive standard library are freely available in source and binary form for all major platforms from the Python Website, and may be freely distributed. The Python Website contains distributions of and pointers to many free third party Python modules, programs and tools, and additional documentation.

Python underwent a major revision from Python 2 to Python 3 in around 2008. There were changes that were not backwards compatible. As a result a lot of Python 2 code needed changing/fixing (refactoring) in order to work in Python 3 environments. This somewhat divided the community. Python 2 is no longer supported, so reported issues with Python 2 are unlikely to be fixed. On the 22nd of August 2022 Python 3.10.6 was released whilst Python 3.11 was in a pre-release testing phase and Python 3.12 was under development. The language development is somewhat open, so developers can get involved and information is provided about development proposals and the status of proposed enhancements. With each main release of Python 3 there is a "whatsnew" Web page that summarises and links to details of the changes:

  • Whatsnew Python 3.0
  • Whatsnew Python 3.1
  • Whatsnew Python 3.2
  • Whatsnew Python 3.3
  • Whatsnew Python 3.4
  • Whatsnew Python 3.5
  • Whatsnew Python 3.6
  • Whatsnew Python 3.7
  • Whatsnew Python 3.8
  • Whatsnew Python 3.9
  • Whatsnew Python 3.10

On whatsnew Web pages, changes are marked against Python Enhancement Proposals (PEPs). These are community proposed and have their own website:https://peps.python.org/

It is important to know exactly which version of Python you are using and what versions of any third party modules you are using. This helps with reproducing results and diagnosing issues.

Checking for Python and installing Anaconda

Check if Python is avalable on your system: start a terminal or command window and enter: "python --version"

That should report the version of python on your system if it is found or report an error.

Please download and install Anaconda or use the distribution available to you. If installing, then choose the option to add Python to your PATH when prompted, and use the default for all the other installation options.

First steps

The getting started tutorial linked from the https://www.python.org/ is a good place to start learning Python. We will be covering a lot of the same things in this course, but we will be going about it in a more geographical way. If you are being tutored at university, then please keep working with these learning resources for now.

Python instructions or commands are entered at a prompt or passed in files to be interpretted. Commands entered at a prompt are run one at a time. When program files are run, typically multiple instructions are run. Let us get going now with a bit of Python coding practise.

Start a terminal or command window and enter the command:

python

The prompt should change to look more like:

>>>

This is the Read-Eval-Print Loop (REPL) environment for Python where user inputs are read and evaluated, and then the results are returned to the user. This is commonly called "the Python prompt" and the terminal or command window the "console".

Enter:

2 + 3

You should see the following printed in the console:

5

Have a play and use this Python REPL environment to do some mathematical calculations.

The following is used for multiplication:

*

Try multiplying two numbers.

The following is used for division:

/

The following is used for raising to a power:

**

Have a go at calculating some larger and smaller numbers.

Many operators can also be accessed via a function in the "operator" module.

Enter: 

operator.add(2, 3)

You should get a "NameError" informing you that the interpretter does not know about the module named "operator".

To use functions in the "operator" module we first have to import the "operator" module! Before doing that, check what modules your Python environment does know about by entering: 

dir()

This runs (or calls) the "dir" function. (You can tell it is a function call as after the name there is a start parenthesis "(". The function call is completed by the end parenthesis ")"). The "dir" function takes no arguments - things that are passed to the function - as there is nothing between the start and end parentheses. (If multiple arguments are passed to a function they are separated by a delimeter - a comma (","))

You should get a list printed out that looks like: 

['__annotations__', '__builtins__', '__doc__', '__loader__', '__name__', '__package__', '__spec__']

You can tell this is a list as it starts with "[" and ends with "]" and the elements are separated with a commas (",") - well, in printing these out it appears as commas and spaces.

Enter: 

import operator

This imports the "operator" module.

Enter: 

dir()

You should now see the "operator" module in the list.

Enter: 

operator.add(2, 3)

You should see 

5
printed in the console.

Enter: 

dir(operator)

This lists all the functions in the "operator" module. It is quite a long list and contains "add", the function that we called to add the numbers 2 and 3. There are numerous functions that have a name starting and ending with "__". These are not supposed to be called directly, but they can be called.

Enter: 

operator.__add__(2, 3)

Python does not have a way of exerting access control on variables or functions, so it uses a naming convention to indicate to users that the variable or function should not be accessed from outside the module. Variables with names that start with a single underscore ("_") or double underscore "__" are generally not supposed to be used directly.

Enter: 

help(operator.add)

This prints out some information about how to use the "operator" module "add" function.

Enter: 

help()

This enters the help system and the prompt changes to look like

help

Read what is printed.

Enter: 

modules

Enter: 

operator

This is the help documentation for the operator module. It is many lines long. Press and hold the <enter> key to get the next lines of the help documentation and let go after a few lines. Try pressing your <space bar> key to get the next page of the help documentation. Usually you can keep pressing your <space bar> key to scan through this help documentation until you exit. You can also press your <q> key to quit and return to the help system.

Enter: 

q

This should exit the help system and return you to the Python prompt.

Call the "quit" function to quit the Python REPL i.e. Enter: 

quit()

This should return you to the normal command prompt.

Next, let us put some commands in a file and run the file.

Open a text editor, add the following lines, then save the file with the name "test.py":

import operator
print(operator.add(10, 15))

From the normal command prompt run: 

python test.py

This should run the program and the output should be displayed in the terminal or command window.

Let's try running Python a third way, in a "Jupyter notebook"

In the normal terminal or command window enter the following at the command prompt: 

jupyter notebook

That should open up a browser running Jupyter with the "Files" tab open. The other tabs are "Running" and "Clusters".

In the "Files" tab select 

New > python 3

That should open up a new browser tab with a Jupyter notebook with the name "Untitled".

Enter the following into the box with the "In [1]" prompt:

import operator
print(operator.add(10, 15))

Click the "Run" button and watch what happens.

Look back at the main Jupyter notebook tab and then return to this notebook.

Click "File > Save as" then enter a filename "test".

Close the browser Window and Click "Quit" in the main Jupyter notebook tab.

Look back at the terminal or command window. Jupyter should be "Shut down" or "Shutting down". If it gets stuck then interupt or kill the terminal or command window and start a new one. Run:

Jupyter notebook
Load "test.ipynb" - the notebook you just created.

Have a play with the 3 different ways of running Python you have just learned about so that you become more familiar with them.
