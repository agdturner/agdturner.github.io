Welcome!

The course Web pages are Open Educational Resources developed by Andy Turner and used for teaching at The University of Leeds. The pages support learning the basics of computer programming in general and the Python programming language (Python) in particular.

In the header of each page is a navigation section which links to the other pages. The navigation section is essentially the same for all the web pages. The link for the currently loaded page has a thin green outline. After the navigation section is a level 1 heading containing the title of the page.

At the bottom of each page there is a footer section which links to the next page (until the last page - a page of references). The page before last is an index page which links back to sections of pages a bit like the index of a text book.

Please go through the pages in order pausing to think, explore links, take breaks, perhaps talk with peers, and undertake practical exercises as directed. It may help you to repeat read some sections or pages before moving on.

There are many links to other websites in these pages, including Wikipedia articles about programming concepts and Python documentation. Don't worry, you don't need to fully understand things before moving on. Try not to get lost in too much detail as you read.

Reading programming language documentation can be a very good way to learn about any language, especially if you are familiar with other programming languages. The style and layout of the documentation for any particular language may take a bit of getting used to, but it is worth getting familiar with this as you practise.

Learning new things is tiring and can be very difficult or impossible to do if you are tired. Try to keep this in mind and manage your time to make good use of your special learning abilities when you have them. Take breaks, look after yourself and try to enjoy the learning journey.

Syllabus

  • Variables
  • Control flow(conditionals, for loops)
  • Classes, functions/methods
  • Testing and debugging (with print, doctest, and unittest)
  • Commenting source code and producing standard documentation of functions, classes and files.
  • Data input and ouput.
  • Data structures and collections.
  • Processing and visualising 2D raster data stored in lists and point data in .
  • Producing documentation for software end users and API documentation using Pydocand Sphinx.
  • Using third party Python packages.
  • Python language development and limitations.
  • Developing a basic Graphical User Interface (GUI)

The Learning Journey

Learning to program involves practical problem solving and computational thinking (such as breaking a problem into a set of smaller tasks). The main workflow is to: plan, write, test, and review.

You are encouraged to be experimental and take your time to develop your understanding and practise your skills. There is a lot to learn especially if you are new to programming.

Following instructions, you are encouraged to develop a simple Agent Based Model (ABM) that represents animals moving about in an environment, interacting with each other and the environment. You are shown how to animate the model and develop a basic GUI so users can run the model with different parametrers without having to modify the source code.
