Introduction to Python Programming for Geography and Social Science

Introduction

Welcome!

This is the home page of a set of web pages that are Open Educational Resources developed by Andy Turner and used for teaching at The University of Leeds. The web pages are for learning the basics of computer programming as might be applied in geography and social science using the Python programming language (Python). At university it is normal to go through these resources on a course with others and with the help of a tutor. In classroom teaching, a tutor may provide practical demonstrations, lead discussion sessions and organise students to work together, discuss and explain things to each other. Those learning remotely are likely to use other means to interact with a tutor and peers. However, these resources can also be followed by anyone wanting to 'go it alone' and teach themselves.

In the header of each web page is a navigation section which links to all the other web pages. The navigation section is essentially the same for all the web pages. To help know which page you are on, the link for that page has a solid outline, and in the main section there is a level 1 heading containing the title of the page. At the bottom of each web page there is a footer section which links to the next page until you are on the last page - a page of references. The page before last is an index page which links back to sections of web pages to help look things up within the pages.

At the top of each page is a Style Button that can be used to change the style of the pages between 'light mode' and 'dark mode'. When active, and by default, 'light mode', sets the background of each page white. Contrastingly, 'dark mode', sets the background of each page black. Other things may be restyled to show up on the background.

The idea is that learners proceed by reading through the web pages in order and pausing to have breaks and to undertake practical exercises as directed. There is extra detail that might be best to skim or skip over the first time you read through a web page. Some detail might be best not to return to until after you have read through more or all the web pages.

Learning new things is tiring and can be very difficult or impossible if you are tired. When you are tired, but not too tired, you may be able to use your coding skills to do things that you have already learned how to do. So, try to keep this in mind and manage your time and make good use of your special learning abilities when you have them. And very importantly - remember to take breaks, rest and recouperate and look after yourself.

Expectations

You will practice developing Python code to:

  • Input data into programs from files and from the Web
  • Output data to files.
  • Use data structures to process coordinates.
  • Display spatial data using matplotlib.
  • Process raster data.
  • Develop and visualise a dynamic spatial simulation controlled from and displayed within a Graphical User Interface.

The Learning Journey

Learning to program involves practical problem solving and computational thinking. This can involve breaking a problem down into smaller tasks and repeating a basic workflow to accomplish each task. The workflow steps are: plan, write, test, and review.

Take time to develop your understanding and practise your skills. There is a lot to learn especially if you are new to programming.

Be experimental and remember to test your understanding and test that your code does what you expect it to do.

The web pages with names starting "ABM" are a series of practicals that develop an Agent Based Model. It is a simplistic model representing animals moving about in an environment, interacting with each other and the environment.
