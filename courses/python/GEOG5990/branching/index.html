Branching

Control Flow

A linear flow of a program that executes one line at a time say from top to bottom would involve a lot of repeated code. A more sophisticated flow reduces the amount of repetition and allowing different threads of execution to occur simultaneously allows for more interactivity with users.

Control Flow Statements

A simple control flow is to branch based upon a condition. So, if the condition is True, then the flow goes one way, otherwise it goes another way. Branching can be done with If Statements and function calls. Since Python 3.10 there is also a Match-case Statement which saves using long chains of If-elif constructs which have multiple conditions that are slightly more efficient than nested If Statements. Consider the following code snippet:

def do_a():
    print("a")
def do_b():
    print("b")
x = 5
y = 0
# A simple If Statement
if x < 0:
    y = -1
    do_a()
# An If-else Statement
if x < 0:
    y = -1
    do_a()
else:
    y = 1
    do_b()
# An If-elif Statement
if x < 0:
    y = -1
    do_a()
elif x == 0:
    y = 0
elif x < 1000:
    y = 1
    do_a()
else:
    y = 2

There can be many elif parts, but since Python 3.10 there is a Match-case Statement which offers a better way to do this.
